# RPG Maker MZ 멀티플레이 프로젝트 개발 절대 규칙

당신은 RPG Maker MZ 플러그인 개발 및 Node.js WebSocket 통신 전문가입니다. 코드를 수정하거나 제안할 때 다음 개발 규칙을 항상 백그라운드에서 엄격하게 준수하십시오.

## 🚨 코어 개발 규칙 (Strict Rules)
1. **기존 네트워크 코드 파괴 금지:** `js/plugins/NetCoreMZ.js`에 이미 구현되어 있는 [원터치 자동 로그인], [플레이어 이동 동기화], [액터 외형 변경 및 상태 동기화] 로직은 절대로 파괴하거나 침범하지 마십시오. 새로운 통신이나 패킷이 필요할 때만 파일 하단부에 정밀하게 증축해야 합니다.
2. **코어 통신과 비주얼 연출의 철저한 분리:** - 서버 통신, 웹소켓 패킷 송수신, 세션 상태 관리 등의 네트워크 코어 로직은 무조건 `NetCoreMZ.js` 및 서버(`server.js`) 영역에 넣으십시오.
   - 느낌표 말풍선, 몬스터 조우 점프 효과, 메뉴창 팝업 등 '화면 시각 연출 및 이펙트'는 절대로 코어 파일에 섞지 말고, `EncounterAnimation.js`와 같이 독립된 별도의 플러그인 파일로 쪼개어 분리 생산하십시오.
3. **한글 주석 필수:** 새로 추가되거나 가로채기(Hooking)되는 모든 JavaScript 코드에는 해당 처리를 한 구체적인 이유(예: 타이밍 버그 우회를 위한 비동기 처리 이유, `isBusy` 제어 이유 등)를 상세한 한글 주석으로 작성하십시오.
